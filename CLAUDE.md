# Cinder UI

Phoenix component library modeled after shadcn/ui.

## Project Structure

- `lib/cinder_ui/` — the library source (components, icons, hooks, classes)
- `priv/templates/` — **source of truth** for `cinder_ui.js` and `cinder_ui.css`
- `demo/` — Phoenix demo app (uses the library as a local dep)
- `dev/` — docs site components, catalog, static docs assets
- `dev/assets/docs/static_docs.js` — vanilla JS for static HTML export (not LiveView)

## Critical: JS and CSS Source Files

**Always edit `priv/templates/cinder_ui.js` and `priv/templates/cinder_ui.css`.**

The files at `demo/assets/js/cinder_ui.js` and `demo/assets/css/cinder_ui.css` are **gitignored copies** generated by `mix cinder_ui.install`. They are overwritten on every `mix assets.build`. Never edit them directly — your changes will be lost.

After editing `priv/templates/cinder_ui.js`, run `mix assets.build` from the `demo/` directory to regenerate the copies and rebuild.

## Build Commands

From `demo/`:
- `mix assets.build` — install cinder_ui assets, compile, build tailwind + esbuild
- `mix phx.server` — run the demo app
- `mix demo.site.export` — export static docs site

From root:
- `mix compile` — compile the library
- `mix test` — run library tests
- `bin/bootstrap` — full project setup (mise, deps, assets, lefthook)

## Component Architecture

Components live in `lib/cinder_ui/components/` grouped by category:
- `actions.ex` — button, toggle, etc.
- `forms.ex` — input, select, autocomplete, label, etc.
- `overlay.ex` — dialog, drawer, sheet, popover, dropdown
- `navigation.ex` — tabs, pagination, breadcrumb, menu
- `feedback.ex` — alert, badge, flash, progress, spinner
- `layout.ex` — card, panel, separator, skeleton, etc.
- `data_display.ex` — table, avatar, chart
- `advanced.ex` — combobox, calendar, carousel, command

LiveView hooks are in `priv/templates/cinder_ui.js`. Each component that needs client-side behavior has a corresponding `Cui*` hook (e.g., `CuiSelect`, `CuiAutocomplete`, `CuiCombobox`).

## Icons

`CinderUI.Icons.icon/1` wraps the optional `lucide_icons` dependency. Use Lucide icon names in kebab-case (e.g., `"chevron-down"`, `"check"`, `"x"`).
